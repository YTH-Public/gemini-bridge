# Repo Name Recommendation

제 추천은 `agent-bridge`입니다.

이유는 세 가지입니다.

1. `gemini`나 `codex`처럼 특정 모델 이름에 묶이지 않아서 앞으로 대상이 더 늘어나도 이름이 안 틀어집니다.
2. 지금 설명하신 구조가 "Claude Code에서 다른 에이전트/Gemini/Codex에게 메시지를 보내는 브릿지"에 가깝기 때문에 의미가 가장 넓고 정확합니다.
3. `ai-bridge`보다 범위가 덜 추상적이라서, 실제로 "에이전트 간 브릿지"라는 용도가 더 잘 드러납니다.

후보별 평가는 아래처럼 봤습니다.

- `agent-bridge`: 가장 균형이 좋습니다. 확장성, 의미 전달, 제품명 독립성 면에서 가장 무난합니다.
- `antigravity-bridge`: Antigravity IDE와 강하게 연결하고 싶다면 좋습니다. 다만 나중에 IDE 바깥에서도 쓰이기 시작하면 이름이 좁아질 수 있습니다.
- `ai-bridge`: 짧고 쉽지만 너무 일반적입니다. 검색성이나 정체성 면에서 약합니다.
- `claude-bridge`: 시작점이 Claude라는 건 드러나지만, 실제로는 Claude와 다른 에이전트들을 잇는 구조라 이름이 한쪽에 치우쳐 보입니다.

추가로 하나만 더 제안하면 `agent-mesh`도 괜찮습니다.

- 브릿지보다 조금 더 네트워크적이고 다중 연결 느낌이 납니다.
- 다만 현재 구조가 정말 "파일 기반 1:1 또는 허브형 브릿지"에 가깝다면 `agent-bridge`가 더 직관적입니다.

정리하면:

- 가장 추천: `agent-bridge`
- Antigravity 브랜드를 더 전면에 두고 싶으면: `antigravity-bridge`

제가 하나를 바로 고른다면 `agent-bridge`로 가겠습니다.
